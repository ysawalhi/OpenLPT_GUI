BUG DETAILS
===============================================================================

BUG-1 (Line 434): RMSE Calculation
-----------------------------------
Current:  self._last_proj_rmse = sqrt(proj_sq / proj_n)
Problem:  proj_n = 2 * N_observations → RMSE underestimated by √2
Fix:      Use proj_n // 2 or track observation count separately
Impact:   Users base decisions on artificially low RMSE (1.4× too optimistic)

BUG-2 (Line 351): Plane Normal Reference
-----------------------------------------
Current:  pt = pt0 + d * n0  (uses reference normal)
Problem:  Should use updated normal 'n' after rotation
Fix:      pt = pt0 + d * n
Impact:   Sub-millimeter for small angles, 3-5mm for large corrections

BUG-3 (Line 543): Success Flag Semantics
-----------------------------------------
Current:  "success": bool(int(res.status) >= 0)
Problem:  status=0 (max_nfev) is NOT convergence success
Fix:      Use res.status > 0 or res.success
Impact:   Downstream may proceed with non-converged calibration

BUG-4 (Lines 150-154): Redundant Initialization
------------------------------------------------
Current:  cv2.solvePnP() → result discarded
Problem:  calibrateCamera() recomputes pose internally
Fix:      Remove solvePnP call entirely
Impact:   Wasted computation, no functional difference

PREC-2 (Lines 420-431): Barrier Function Smoothing
---------------------------------------------------
Current:  if gap > 0: [nonzero residual] else: [0.0, 0.0]
Problem:  C⁰ discontinuity at gap=0 creates gradient kink
Fix:      Use softplus: gap_smooth = tau * log1p(exp(gap/tau))
Impact:   Smoother optimization landscape, better convergence

PREC-3 (Line 503): Parameter Scaling
-------------------------------------

Problem:  Heterogeneous units (mm/rad/relative) unscaled
Fix:      Add x_scale='jac' and diff_step array per parameter type
Impact:   Better conditioning, fewer optimizer iterations
